## Architecture

```
CEO Olly Henson (Human — Final Authority)
  |
  +-- AGENT.md (The Brain)
  |     The master instruction set. Every AI interaction starts here.
  |     Contains: identity, audience, voice, pillars, offers, ethics.
  |
  +-- Skills (The Workers)
  |     Each skill file is a specialist that knows how to do one job.
  |     All skills defer to AGENT.md for voice, audience, and rules.
  |
  |     +-- skills/brand-voice.md
  |     +-- skills/content-strategy.md
  |     +-- skills/content-formula.md
  |     +-- skills/instagram-strategy.md
  |     +-- skills/youtube-strategy.md
  |     +-- skills/blog-seo-guide.md
  |     +-- skills/email-marketing.md
  |
  +-- Memory (The Context)
  |     Persistent files that accumulate knowledge over time.
  |     Skills read from memory to make better decisions.
  |
  |     +-- memory/content-log.md      (What's been created)
  |     +-- memory/performance.md      (What's working)
  |     +-- memory/calendar.md         (Key dates and plans)
  |     +-- memory/audience-insights.md (What we know about the audience)
  |
  +-- Configuration
  |     System-level settings that govern how the OS operates.
  |
  |     +-- connections.md    (Tech stack integrations)
  |     +-- governance.md     (Auto vs Ask rules)
  |     +-- schedule.md       (When things happen)
  |     +-- org-chart.md      (This file — the map)
  |
  +-- Guides
        Setup and reference documentation.

        +-- QUICK_START.md    (Five-minute setup)
        +-- README.md         (What this is, how to use it)
```

## How It Flows

### Content Creation Flow

```
1. schedule.md triggers content creation (weekly rhythm)
2. AGENT.md provides brand context and voice rules
3. content-strategy.md defines what to create (pillars, platforms, mix)
4. content-formula.md provides structures and templates
5. [platform skill] adapts for the specific platform
6. governance.md determines if approval is needed
7. content-log.md records what was created
8. performance.md tracks how it performed
```

### Decision Flow

```
AI wants to take an action
  |
  +-- Is it in governance.md AUTO list? -> Proceed, log it
  |
  +-- Is it in governance.md ASK list? -> Present to CEO Olly Henson, wait
  |
  +-- Is it in governance.md NEVER list? -> Do not attempt
  |
  +-- Not listed? -> Default to ASK
```

## The CEO Agent (Future — Level 5)

When 2+ departments are active, a CEO Agent sits above them all:

```
CEO_AGENT.md (Governor — not a worker)
  |
  +-- Marketing OS (this department)
  +-- Operations OS
  +-- Sales OS
  +-- ... (additional departments)
```

The CEO Agent:
- Sees activity across all departments
- Makes strategic decisions (which department gets priority)
- Escalates to CEO Olly Henson for high-stakes decisions
- Runs weekly reviews
- Manages cross-department handoffs
- Enforces governance standards
